数据的最小单位：字段（Field）

 

 

一个字段就是一格数据。常见类型：

 

Data（字符串）：「客户名称」

Int/Float（数字）：「数量」「重量」

Currency（金额）：「含税金额」

Date/Datetime（日期时间）

Check（勾选/布尔值）

Link（外键，指向另一张表的一条记录）

Select（下拉：固定选项）

Table（子表，承载“多行明细”，每行又是一条“子记录”）

Attach/Image（文件/图片）

JSON/Code/Text（长文本/结构化文本）

你在界面上看到的每个输入框，本质都是一个 Field。

 

 

Doctype

就是一张表的结构➕规则

有哪些Field， 类型，是否必填，唯一？ 默认？ 校验规则？提交？树形？

比如： customer 这个doctype 就有name phone addr 等等

 

Doc = 某个 DocType 的一条记录（表里的一行）

 

常见doctype类型

 

 

Standard DocType（标准）

 

最普通的一张表。每创建一条 Doc，DB 多一行。

例：Customer、Item、Sales Invoice。

Child Table DocType（子表）

 

专门用来做明细行的“表结构”。不能单独使用，只能被主表的 Table 字段引用。

例：Sales Invoice Item 是 Sales Invoice 的明细行；DB 表是 tabSales Invoice Item。

Single DocType（单例设置）

 

没有独立表，所有字段都放进系统的 tabSingles 表里。

典型是“系统设置类”数据，如 Global Defaults、Company 设置页。

这类是“一份配置，一个页面”，不是很多行。

标准操作流程

 

RPNext 的核心思想是：从计划到下单，再到收货和付款，逐步转化。

 

Material Request（物料需求单）

→ 计划部门或仓库发起需求。

（告诉系统“我要买什么”）

Supplier Quotation（供应商报价单）

→ 向供应商询价（可选）。

Purchase Order（采购订单）

→ 确认要采购，正式下单。

✅ 关键字段：Supplier、Item、Qty、Rate、Company。

Purchase Receipt（采购收货单）

→ 仓库收货时录入。

✅ 自动更新库存数量。

📦 一般来自 Purchase Order。

Purchase Invoice（采购发票）

→ 财务收到发票后创建。

✅ 产生应付款（Creditors）。

✅ 连接 Purchase Receipt。

Payment Entry（付款记录）

→ 向供应商付款。

✅ 结清应付款。

 

 

 

📦 二、销售 / 出货流程（Sales Cycle）

从客户下单 → 仓库发货 → 开发票 → 收款。

 

 

🔹 流程顺序：

 

 

Quotation（报价单）

→ 给客户报价。

Sales Order（销售订单）

→ 客户确认购买。

✅ 系统锁定库存。

Delivery Note（出货单）

→ 仓库发货。

✅ 扣减库存数量。

Sales Invoice（销售发票）

→ 财务开票。

✅ 生成应收账款（Debtors）。

Payment Entry（收款记录）

→ 收款，结清应收款。

 

 

 

👷‍♀️ 三、工资 / 薪资流程（Payroll Cycle）

 

 

 

🌐 关键概念

 

 

工资模块不是简单的“发钱”，而是：

 

Payroll Structure（薪资结构）

Payroll Entry（工资单汇总）

Salary Slip（单人工资单）

Assignment（分配规则）

 

 

 

🔹 标准流程：

 

 

Employee（员工）

→ 建立员工档案。

Payroll Structure（薪资结构）

→ 定义工资组成，比如：

 

Basic（基本工资）

Allowance（补贴）

Tax（扣税）

Superannuation（养老金）

Assignment（工资结构分配）

→ 指定哪位员工用哪种结构。

（比如普通工 vs 经理）

Salary Slip（工资单）

→ 每位员工生成月度工资单。

✅ 可自动计算工时、扣税、津贴。

Payroll Entry（工资发放汇总）

→ 批量生成所有员工工资单。

✅ 自动记账（Debit Expense / Credit Payable）。

Payment Entry（发放工资）

→ 实际发钱。

 

 

 

🧰 四、资产录入 / 折旧（Fixed Assets）

 

 

 

🌐 流程逻辑

 

 

资产模块在 ERPNext 里是自动与会计账簿、折旧计划结合的。

 

 

🔹 流程顺序：

 

 

Purchase Invoice（采购发票）

→ 购入资产时选择 Asset Item。

✅ 自动生成 Asset 记录。

Asset（资产）

→ 包含：

 

资产编号

购入日期

折旧方法（Straight Line / Double Declining）

折旧年限

Depreciation Schedule（折旧计划）

→ 系统自动生成。

Depreciation Entry（折旧分录）

→ 按期自动过账。

✅ Debit: Depreciation Expense

✅ Credit: Accumulated Depreciation

Asset Disposal / Sale（资产处置）

→ 出售或报废资产。

✅ 生成 Gain/Loss。
